# 📝 Focusdo App

[English](./README.md) | [한국어] | [日本語](./README.ja.md)

---

일일 투두 관리와 집중 타이머를 결합한 생산성 애플리케이션입니다.  
이 프로젝트는 상태 책임을 명확히 분리하고, 예측 가능한 UI 흐름과
확장 가능한 프론트엔드 아키텍처를 설계하는 데 초점을 두고 있습니다.

단편적인 생산성 도구를 전환하며 사용하는 대신,  
의도적인 집중을 기반으로 하루의 작업을 관리하고자 하는 사용자를 위해 설계되었습니다.

---

## ✨ 주요 기능

- 📅 날짜 기반 Todo 관리
- ⏱️ 프리셋 및 사용자 지정 집중 타이머
- 🧠 작업별 집중 시간 자동 누적
- 🚫 과거 / 미래 날짜에 대한 읽기 전용 모드
- 📊 일일 대시보드 (완료율, 총 집중 시간, 완료 작업 수)
- 📈 집중 히스토리 뷰 (일간 / 주간)
- ⚠️ Loading / Empty / Error UI 상태 명시적 처리

---

## 🗂️ 페이지 구성

- **Today Page**  
  일일 Todo를 추가, 수정, 관리하는 메인 페이지이며  
  대시보드 요약 정보를 함께 제공합니다.

- **Focus Page**  
  선택한 Todo에 대해 집중 세션을 실행하는 전용 페이지입니다.

- **Focus History Page**  
  주간 집중 기록은 차트로 시각화하고,  
  일간 기록은 요약 카드와 상세 모달 구조로 제공합니다.

## 🛠️ 기술 스택

- **React + TypeScript**
- **Zustand** — 전역 상태 관리
- **TanStack Query** — 비동기 상태 및 로딩/에러 처리
- **React Router**
- **Tailwind CSS**
- **shadcn/ui** — 접근성을 고려한 UI 컴포넌트
- **Day.js** — 날짜 유틸리티

## 🧩 아키텍처 설계

상태 흐름은 다음과 같이 단방향으로 설계되어 있습니다.  
사용자 인터랙션 → 페이지 오케스트레이션 → 스토어 변경 → 파생된 UI 렌더링

- **페이지 단위 오케스트레이션**  
  페이지 컴포넌트는 도메인 로직을 직접 구현하지 않고,  
  스토어와 커스텀 훅을 조합하여 UI 흐름을 조정합니다.

- **도메인 로직 분리**  
  Todo 및 집중 시간 관련 비즈니스 로직은  
  store와 custom hook 내부에 분리되어 있습니다.

- **명시적인 UI 상태 모델링**  
  Loading, Empty, Read-only, Error 상태를 의도적으로 구분해 렌더링합니다.

- **과도한 최적화 지양**  
  리스트 가상화나 무한 스크롤은  
  실제 사용 패턴이 확인된 이후 도입할 예정입니다.

## 🧠 디자인 결정 사항

- 읽기 전용 날짜는 명시적인 UI 상태로 모델링하여
  오해의 소지가 있는 인터랙션을 방지합니다.
- Empty 상태는 현재 날짜 컨텍스트에서
  사용자 입력 가능 여부를 반영하여 렌더링됩니다.
- 날짜 네비게이션 로직은 상태를 가지지 않도록 설계해
  페이지 간 숨은 결합을 방지했습니다.

## ⚖️ 알려진 절충 사항

- 모든 데이터는 현재 메모리 기반으로 관리됩니다.
